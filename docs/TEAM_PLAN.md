# План разработки MedArchive — распределение по команде

Документ для раздачи участникам. Отражает реальное состояние на 27.06.2026.
У каждой задачи: владелец, зависимости, что сделать, критерий «готово».

---

## Что уже готово (не трогаем, только используем)
- **БД**: схема (`sql/schema.sql`) развёрнута в Supabase, индексы + `pg_trgm` + дедуп-индекс.
- **Парсеры**: PDF-текст, PDF-скан (OCR), XLSX (мультилист), DOCX; ZIP-распаковка,
  определение типа файла, валидация, CLI с выгрузкой в JSON/CSV.
- **Нормализатор** (`normalizer/`): каскад точное→синонимы→аббрев→fuzzy→семантика,
  два порога, blocking, ловушки, оценочный стенд (`eval/`), read-слой `repository.py`.
  Проверен на живом справочнике: synonyms/exact/fuzzy ловятся; семантика требует
  установленных зависимостей (см. задачу N).

## Задача 1. Загрузчик парсер → Supabase  **(БЛОКЕР, делать первым)**
**Владелец:** ___  **Зависит от:** ничего  **Приоритет:** 🔴 высокий

Сейчас `repository.py` умеет только читать. Парсер пишет в JSON/CSV, в базу данные
не попадают — пайплайн разорван.

- Добавить write-функции в `repository.py`:
  `upsert_partner()` (дедуп по `bin`, иначе `name`+`city`),
  `insert_document()` → `doc_id`, `bulk_insert_items()` (учёт `uq_item_dedup`).
- Транзакция на документ; ошибка → откат + статус `error`. Идемпотентность.
- Этап 0: привести `models.PriceItem` к схеме (добавить `service_id`,
  `match_confidence`; `source_row`/`source_sheet` не писать в БД).
- В `cli.py` флаг `--to-db`.

**Готово, когда:** прогон архива наполняет `partner`/`price_document`/`price_item`
в Supabase, повторный прогон не дублирует.

## Задача 3. Версионирование, верификация, аномалии (ТЗ 4.4, вес 20%)
**Владелец:** ___  **Зависит от:** Задача 1  **Приоритет:** 🟡 средний

- При изменении цены: старую версию `is_active=FALSE` (не удалять), новую — `TRUE`.
- Аномалия >50% к предыдущей версии → `needs_review` + `verification_note`.
- `POST /verify` (подтвердить/отклонить/исправить); подтверждение → `repository.add_synonym`.
- Использует write-слой `repository.py` из Задачи 1 (+ функция «найти предыдущую версию»).

**Готово, когда:** повторная загрузка с новой ценой архивирует старую и метит аномалии.

## Задача 6. Нормализатор — доведение до боевого
**Владелец:** ___ (автор нормализатора)  **Зависит от:** ничего  **Приоритет:** 🟢 низкий

- На каждой машине/сервере поставить зависимости (иначе семантика молча выключается):
  `pip install rapidfuzz "sentence-transformers" pymorphy3`.
  Проверка: в выводе `eval` должно быть `fuzzy=rapidfuzz`, а не `difflib-fallback`.
- Расширить `abbreviations.json` и стоп-слова под реальные прайсы.
- Gold-набор довести до 200–300 пар, откалибровать пороги (`--auto`/`--unmatched`).
- Цель: автосопоставление ≥70% (с эмбеддингами реально 85%+).

**Готово, когда:** на реальном архиве доля автосопоставления ≥70% при precision ≥0.9.
